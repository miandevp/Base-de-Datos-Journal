
# Seguridad en Bases de Datos

## Error de Capa 8

Se refiere a errores causados por las personas y no por la tecnología. El término surge porque en el modelo OSI existen 7 capas, por lo que la "capa 8" representa al usuario.

**Ejemplos:**

* Contraseñas débiles.
* Compartir credenciales.
* Abrir enlaces maliciosos.
* Configurar permisos incorrectamente.

---

## Acceso Programático

Acceso a una base de datos mediante aplicaciones o código en lugar de hacerlo manualmente.

**Ejemplos:**

* Aplicaciones web.
* APIs.
* Scripts en Python, Java o C#.

---

## SQL Dinámico

Técnica que permite construir consultas SQL durante la ejecución del programa.

**Ventajas:**

* Flexibilidad.

**Riesgos:**

* Vulnerabilidades de SQL Injection si no se valida la entrada del usuario.

---

## Stored Procedure

Conjunto de instrucciones SQL almacenadas dentro de la base de datos que pueden ejecutarse cuando se necesiten.

**Ventajas:**

* Reutilización.
* Mejor control de acceso.
* Centralización de lógica.

---

## Caso de Uso

Escenario que describe cómo un usuario o sistema interactúa con la base de datos para cumplir un objetivo específico.

**Ejemplo:**

* Registrar un nuevo cliente.
* Consultar el historial de compras.
* Actualizar información de un producto.
